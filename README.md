# 문·창문 개방 감지 스마트 에어컨 제어 시스템

uC/OS-III 기반의 실시간 임베디드 시스템으로, 문·창문 개방 상태와 실내 온습도 정보를 함께 판단하여 에어컨 냉방 동작을 자동으로 제어한다. 일반 에어컨이 실내 온도만 보고 동작하는 것과 달리, 문이나 창문이 열린 상태에서의 불필요한 냉방과 전력 낭비를 감지·차단하는 상황 인식형 제어를 목표로 한다.

실제 냉방 장치를 구동하는 대신, 2자리 7세그먼트·온보드 LED·외부 부저·UART 출력을 통해 현재 온도, 희망 온도, 냉방 단계(L1~L3), 상태 코드(OP, ES, OF)를 상황별 우선순위에 맞춰 전환 표시함으로써 시스템을 시연한다.

## 개발 배경

냉방 시스템은 사용자의 부주의로 에너지 낭비가 발생하기 쉽다. 대표적으로 문·창문이 열린 상태에서 에어컨이 계속 작동하면, 실내 냉기가 빠져나가고 외부의 더운 공기가 유입되어 에어컨이 더 강하게 동작한다. 결과적으로 전력 소모가 증가하고 냉방 효율은 떨어지지만 사용자는 원인을 즉시 인지하기 어렵다.

본 프로젝트는 이 문제를 **센서 기반 상황 인식**과 **RTOS 기반 실시간 제어**로 해결한다. 문이 짧게 열리는 경우는 일상적인 통행으로 보고 냉방을 유지하되, 일정 시간 이상 개방이 지속되거나 고온다습한 공기 유입이 감지되면 경고를 발생시키고 냉방을 일시 정지한다.

## 주요 기능

- **개방 감지**: 리드 스위치 센서로 문·창문 개폐를 판별하고, 개방 지속 시간을 판단한다.
- **온습도 기반 냉방 제어**: DHT11 센서로 2초 주기 측정, 희망 온도와 비교해 냉방 세기(L1~L3)를 자동 결정하며, 급격한 습도 상승으로 열기 유입을 정밀 판정한다.
- **사용자 입력**: 버튼으로 전원 ON/OFF, 희망 온도 조절, 자동 제어를 우회하는 **수동 제어 우선 모드(Manual Override)**를 지원한다. (하드웨어·소프트웨어 debounce 적용)
- **다중 출력**: 2자리 7세그먼트의 시각적 한계를 극복하기 위한 타임 슬라이싱·전환 표시, 온보드 LED(LD1~LD3) 상태 매핑, 외부 부저 경보를 함께 사용한다.
- **RTOS 기반 동시성 제어**: 6개 task를 message queue, event flag, mutex semaphore로 연동하여 Race Condition과 자원 충돌 없이 동작하도록 설계한다.

## 하드웨어 구성

| 구분 | 장치 | 역할 |
| --- | --- | --- |
| Main Board | NUCLEO-F439ZI | Cortex-M4 MCU, 전체 제어 및 uC/OS-III 구동 |
| 입력 | DHT11 온습도 센서 | 실내 온습도 2초 주기 스캔 |
| 입력 | 리드 스위치 센서 | 문·창문 개방 감지 (EXTI 인터럽트 기반) |
| 입력 | 버튼 모듈 | 외부 택트 버튼 2개(희망 온도 UP/DOWN), 온보드 USER 버튼 B1(수동 모드 토글·OFF) |
| 출력 | 온보드 LED (LD1~LD3) | 상태별 시각 피드백 (녹색: COOLING, 청색: MANUAL, 적색: WARNING) |
| 출력 | 외부 부저 모듈 | 개방 시간 초과 시 경보음 |
| 출력 | 2자리 7세그먼트(FND) | 현재 온도·설정 온도·상태 코드 전환 표시 |
| 통신/디버그 | UART (ST-LINK VCP) | 115200bps 디버그 터미널 로그 출력 |

## 소프트웨어 설계 (uC/OS-III)

### Task 구성

사용자 입력 응답 성능을 위해 User Input Task의 우선순위를 출력 전담 task보다 상향 배치했다. (숫자가 작을수록 높은 우선순위)

| Task | 우선순위 | 역할 |
| --- | --- | --- |
| Sensor Task | 5 | DHT11 2초 주기 스캔, 리드 스위치 감지 이벤트 생성 |
| User Input Task | 6 | 버튼 입력(EXTI) 수신, debounce 후 이벤트 플래그 게시 |
| Control Task | 7 | 센서·사용자 입력 종합, 상태 머신 갱신 및 출력 명령 생성 |
| Display Task | 8 | 7세그먼트 멀티플렉싱 구동 및 우선순위별 화면 타임 셰어링 |
| Actuator Task | 9 | LED·부저 GPIO 패턴 구동 |
| Monitor/Debug Task | 12 | UART로 시스템 상태·CPU 점유율·큐 현황 출력 |

### Task 간 통신 및 동기화

- **Message Queue**: Sensor Task가 수집한 센서 데이터(SensorMsg)를 Control Task로 전송하고, Control Task의 출력 패킷(DisplayMsg, ActuatorMsg)을 각 전용 큐로 분리 송신해 표시·경보 장치가 독립적·병렬적으로 동작하게 한다.
- **Event Flag**: User Input Task의 비동기 입력(희망 온도 조절, 수동 모드 토글 등)을 플래그로 게시하여 Control Task가 즉각 대응한다.
- **Mutex Semaphore**: 여러 task가 공유하는 UART 포트를 상호 배제로 보호하여 출력 충돌을 막고, 우선순위 역전을 방지해 실시간성을 보장한다.

## 제어 상태 정의

Control Task의 FSM(유한 상태 머신)이 센서·사용자 입력에 따라 상태를 천이한다.

| 상태 | 조건 | 동작 요약 |
| --- | --- | --- |
| OFF | 전원 OFF | 7세그 `OF` 표시 후 꺼짐, 모든 LED·부저·연산 정지 |
| IDLE | 현재 온도 ≤ 희망 온도 | 현재 온도 표시, LED·부저 OFF |
| COOLING | 현재 온도 > 희망 온도 + 1°C | 현재 온도 + dp 점등, LD1(녹색) 점등, 세기 L1~L3 가동 |
| WINDOW_SUSPECT | 개방 감지 직후 경계 | 현재 온도와 `OP` 1초 간격 교대, LD3(적색) 느린 점멸, 부저 단발음 |
| AC_PAUSED | 개방 5초 초과 또는 습도 10% 급등 | `OF` 0.5초 점멸, 냉방 중지, LD3 고속 점멸, 부저 연속 경보 |
| RECOVERY_WAIT | 개방 종료 후 안정화 5초 | 현재 온도와 `ES` 1초 간격 교대, 경보 해제, 냉방 재가동 보류 |
| MANUAL_OVERRIDE | USER 버튼 진입 | 현재 온도 + dp 점등, LD2(청색) 점등, 자동 개방 감지·정지 무시 |

## 기대 효과

온도만 보는 시스템은 "왜 온도가 내려가지 않는지" 알 수 없지만, 본 시스템은 문·창문 개방 상태를 함께 판단해 냉방 효율 저하의 원인을 감지한다. 사용자에게 즉시 경고를 제공하고 장시간 개방 시 냉방 출력을 줄여 에너지 절약 효과를 기대할 수 있으며, 실제 스마트홈에서 활용 가능한 에너지 절약형 자동 제어 구조를 소형 임베디드 시스템으로 구현한다.
